# Homework TP-3

## 1. Проект состоит из двух частей
Генератор данных создает CSV-файл `data.csv`.
Аналитик данных читает файл, созданный генератором, и формирует на его основе HTML-отчет `report.html`.
Все основные действия выполняются через скрипт `run.sh`.

## 2. Структура репозитория

`run.sh` — главный скрипт проекта. Через него выполняется сборка и запуск контейнеров, создание локальных данных, просмотр структуры проекта, очистка папки `data`, проверка содержимого `data` внутри контейнеров и запуск веб-сервера для отчета.
`data/` — папка для результатов работы контейнеров. В ней появляются `data.csv` и `report.html`.
`generator/` — папка с генератором данных. Внутри находится Python-скрипт и Dockerfile.
`reporter/` — папка с аналитиком данных. Внутри находится JS-скрипт, Dockerfile и описание Node.js-проекта.

## 3. Запуск проекта

Сначала нужно дать файлу `run.sh` право на выполнение:
```bash
chmod +x run.sh
```

Собрать и запустить генератор:
```bash
./run.sh build_generator
./run.sh run_generator
```
После этого в папке `data` должен появиться файл `data.csv`.

Сборка и запуск аналитика:
```bash
./run.sh build_reporter
./run.sh run_reporter
```
После этого в папке `data` должен появиться файл `report.html`.

Проверить результат можно командой:
```bash
ls data
```
В папке должны быть два файла:
```text
data.csv
report.html
```

Дополнительные команды:
```bash
./run.sh structure
./run.sh clear_data
./run.sh inside_generator
./run.sh inside_reporter
```

## 4. Отчет

Для публикации отчета используется контейнер `nginx`. Он нужен, чтобы открыть готовый HTML-файл через браузер. 

Источник, по которому изучала работу nginx: https://hub.docker.com/_/nginx

Перед запуском сервера нужно создать данные и отчет:
```bash
./run.sh build_generator
./run.sh run_generator
./run.sh build_reporter
./run.sh run_reporter
```
После этого сервер запускается командой:
```bash
./run.sh report_server
```
При запуске создается контейнер `nginx`, папка `data` монтируется внутрь контейнера, а порт `80` контейнера пробрасывается на порт `8080`. Локально отчет можно открыть так:
```text
http://localhost:8080/report.html
```
Если проект запускается в GitHub Codespaces, нужно открыть вкладку `Ports`, найти порт `8080` и нажать `Open in Browser`.
Иногда браузер открывает только общий адрес порта. Тогда в конец ссылки нужно вручную добавить:
```text
/report.html
```
